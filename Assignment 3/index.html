☆ Requirement 3 Questions ☆

What do you know about the client?

The client is the device that an end user uses to obtain services or resources from the server, such as a web browser. This can include a smartphone, tablet, laptop, or desktop computer.It is compatible with web browsers including Chrome, Firefox, Safari, and Microsoft Edge. To display web pages and interact with users, the client must interpret the server-sent HTML, CSS, and JavaScript code.

Where can we learn more about the client?

We can learn more about the client provided by our web browsers like google chrome which provides documentation into how the client works with web applications.

What do you know about the server?

The server is the backend component of any web application. While the client connects with the server to access resources or services, the server is responsible for providing and handling client requests, accessing databases, and data processing.

Where can we learn more about the server?

We can learn more about the server by exploring resources related to backend development. There are many online tutorials, documentation from web frameworks including programming languages like Javascript and Python, which can provide information related to server-side structure and implementation.

The interactions among client and server in my web applications and the languages used to define them.

HTTP requests and responses played a major role in enabling client-server interactions in my web application. Through a web browser, the server is responsible for handling HTTP requests from the client. The client asks the server to retrieve resources like HTML, CSS, and JavaScript files. These resources define my web pages structered content, style, and interactive elements/functionality. After recieving the requests, the server was able to process them and send HTTP responses back to the client with the resources needed to display my web page accurately.

The role of HTML in my temperature converter:

The foundation of a web page is its HTML, which establishes its structure and content. HTML is utilized in my webpage to develop the user interface's structure and the layout in my temperature converter implementation. It outlines the components needed for the temperature conversion, including the input number fields, buttons, labels, and containers.

The role of CSS in my temperature converter:

The HTML elements of my webpage are styled with CSS, which enhances the website's visual design and layout. The implementation of my temperature converter uses CSS to control the elements defined by HTML. CSS helped me in adjusting the temperature conversion's colors, fonts, sizes, margins, and alignments to make it more aesthetically pleasing and easier to use.

The role of JavaScript in my temperature converter:

JavaScript is a scripting language that adds interactive elements and functionality to websites. Javascript was essenital in calculating the conversion from Fahrenheit to Celsius and would not have been possible without it. Additionally, JavaScript obtains the information from the user input field, which in this case was the temperature in Fahrenheit. It then does the math required to convert the temperature to Celsius, which then updates the HTML element displaying the result instantly. Lastly, Javascript enabled me to display the current date and time on my webpage by updating the corresponding HTML element.
